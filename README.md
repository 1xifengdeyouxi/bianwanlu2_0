# 便玩录 2.0

> 一个面向个人效率管理的原生 Android 应用，围绕 **笔记 / 待办 / 时间轴 / 日历 / 提醒 / 数据备份** 构建“本地优先”的效率闭环。

## 项目简介

- 这是一个以“记录 + 规划 + 回顾”为目标的原生 Android 项目。
- 项目不仅关注功能可用，还重点关注 **UI 体验、复杂状态管理、背景提醒链路、本地数据安全与模块化架构**。
- 当前版本已具备笔记、待办、分类管理、搜索、时间轴、日历聚合、提醒设置、数据导入导出、个性化设置等核心能力。

## 面试官可快速关注的亮点

- **复杂功能闭环完整**：不是单一 CRUD Demo，而是把笔记、待办、时间轴、日历、提醒、搜索、备份等串成一个真实可用的效率工具。
- **模块化架构渐进改造**：参考 Now in Android 的分层思路，将项目拆分为 `app / core / feature`，在不破坏现有功能的前提下完成模块化演进。
- **背景提醒链路完整**：支持 AlarmManager 精确提醒、开机重排、置顶通知、锁屏提醒、稍后 10 分钟、从通知点击直达目标页面。
- **系统能力融合**：提醒事项可写入系统日历，并提供消息提醒、震动、精确闹钟、后台省电保护的授权引导。
- **本地数据安全设计**：已实现 JSON 备份导入 / 导出，包含 schema version、导入预览、Room 事务恢复、默认分类兜底与提醒重同步。
- **账号数据隔离**：支持游客态与账号态的本地数据隔离，并支持游客数据向登录账号的迁移。
- **复杂 UI 交互状态管理**：包括左滑抽拉操作、批量选择、长按排序、实时搜索高亮、日历聚合展示等较复杂的 Compose 状态场景。

## 功能清单

- **笔记**：列表筛选、新建 / 编辑、图片插入、文字颜色、提醒、优先级、完成状态、批量选择。
- **待办**：进度筛选、完成 / 未完成切换、批量提醒、批量优先级、批量删除、拖拽排序。
- **分类管理**：分类新增、修改、清空、删除、排序、左滑多操作。
- **时间轴**：聚合笔记与待办的新增、修改、删除、完成、提醒行为记录，支持时间范围和分类筛选。
- **日历**：月视图展示提醒分布，支持月份切换、点击查看当日内容。
- **搜索**：实时匹配、历史记录、分类筛选、关键词高亮、从搜索结果直达笔记 / 待办详情。
- **我的 / 设置**：提醒设置、数据与同步、通用设置、关于页面、个人信息等组合界面。
- **个性化**：全局字体大小、单条最大高度、主题模式、皮肤色、头像 / 昵称状态同步。
- **数据与备份**：通过 Android SAF 完成 JSON 导入 / 导出，并提供导入预览、清空本地数据、恢复完成后的提醒重同步。

## 技术栈

- **开发语言**：Kotlin
- **UI**：Jetpack Compose + Material 3
- **架构**：MVVM + StateFlow + Coroutines
- **依赖注入**：Hilt
- **本地持久化**：Room
- **系统集成**：AlarmManager、BroadcastReceiver、NotificationChannel、CalendarContract
- **文件能力**：ActivityResult API + Android SAF
- **本地状态存储**：SharedPreferences + StateFlow 封装

## 工程结构

```text
app

feature
  notes

  timeline
```

### 模块分层说明

- `app`：应用壳层，负责入口、导航、抽屉、搜索、我的与设置等顶层组装。
- `core:common`：轻量公共常量与通用能力。
- `core:model`：实体模型与领域数据结构。
- `core:data`：Room Database、DAO、Repository、提醒协调、备份恢复、账号隔离等数据层能力。
- `core:ui`：主题、字体系统、公共 UI 组件。
- `feature:*`：按功能域拆分的业务模块，目前已完成 notes / todo / category / calendar / timeline 拆分。

> 说明：搜索、我的、设置等跨页面组合逻辑目前仍保留在 `app`，后续会继续下沉。

## 我在这个项目里重点解决的问题

- **复杂状态分层**：把列表筛选、批量选择、排序、交互动效和编辑状态拆分到 ViewModel + UI state 中，减少 Compose 页面耦合。
- **后台提醒可靠性**：使用 `AlarmManager + BroadcastReceiver + NotificationChannel + 重排策略` 构建可落地的本地提醒方案。
- **业务数据与衍生数据一致性**：例如提醒、时间轴、分类状态、日历聚合等衍生展示都需要跟随原始数据及时同步。
- **本地数据迁移与恢复**：在导入 / 导出和账号切换过程中，兼顾默认分类、ID 重映射、提醒重同步和游客数据迁移。
- **工程化演进**：从相对集中的页面结构逐步迁移到 feature modules，在重构过程中优先保证功能不回归。
